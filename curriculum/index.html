Relational Database Concepts

Relational databases organize data into one or more tables or relations. Relation is the formal, academic term. In day-to-day professional work we usually say table, which is the term preferred in this course.

We can imagine a table as a two dimensional grid of cells. Each cell holds one piece of data or one fact. A row in a table is a horizontal layer of cells, one cell high. It holds facts about one discrete thing. That thing could be anything: a person, a credit card transaction, or a professional sports mascot. A column in the table is a vertical strip of cells, one cell wide. Every cell in the column holds the same type of data, but each cell is a fact about a different thing.

In the data above, each row represents a different state. A row is a stand-alone thing. Rows are also known as records or tuples. Record is relatively common, tuple less so -- it's an academic term. Each column represents a fact about a state: its name, abbreviation, capital, data established, and current population.
