Pokémon - React Native

Goal

Build a React Native app that uses a Pokémon API as its data source and tracks favorite Pokémon with React Context or Redux. The app should use React Navigation.

Screens

Home Screen

Should list Pokémon with the ability to page through them in sets of 20. (Start with an initial set of 20 with no paging.) Use the Poké API to fetch Pokémon (https://pokeapi.co/docs/v2#pokemon.) See also examples from https://pokeapi.co/ to limit results to specific pages with limit and offset.

The Home Screen should navigate to the Detail Screen.

The Home Screen can also navigate to the Favorite Screen.

Detail Screen

Should display a single Pokémon with details including type and images. (Go as crazy with details as you like: movement, abilities, forms, evolutions...) Use https://pokeapi.co/docs/v2#pokemon with an id (Dex number) to fetch the full details of a Pokémon.

The Detail Screen should allow a user to mark that specific Pokémon as a "favorite" or remove that specific Pokémon as a favorite. (Toggle favorite.)

Favorite Screen

Should display favorite Pokémon in a quick summary format along with an image. Favorites are tracked with React Context or Redux.

The Favorite Screen should navigate to the Detail Screen.

Optionally, the Favorite Screen should have a way to remove a favorite.

Libraries

  • React Navigation
  • React Context
  • Redux
  • Redux Toolkit

Steps

  1. Create an expo app: npx create-expo-app pokemon.
  2. Add React Navigation: https://reactnavigation.org/docs/getting-started
  3. Start with the Home Screen. Don't worry about paging for now.
  4. Keep it simple. Don't go crazy with styles.
  5. Add the Detail Screen and navigate from the Home Screen, passing the Dex number as a parameter. Keep the Detail Screen relatively simple.
  6. Add paging to the Home Screen.
  7. Implement the favorite toggle with either React Context or Redux (Toolkit).
  8. Create a Favorite Screen.
  9. Navigate from the Home Screen to the Favorite Screen.
  10. Add more details: enrich the Details Screen, remove a favorite from the Favorite Screen, etc.
  11. Add more styling.

Stretch Goals

  • Add additional screens that include abilities, moves, locations, berries, etc, based on the PokéAPI.
  • Persist favorites permanently with a solution like Sqlite or some other local data store.
  • Persist favorites permanently using an API!
