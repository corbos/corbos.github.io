Introduction

Cascading Style Sheets (CSS) is an open standards language used to transform our HTML into colorful, multi-font designs with rich layouts. All modern browsers support CSS. The language is specified by the W3C.

CSS is composed of rulesets. All rulesets include a selector that targets HTML elements. Selectors can be very broad (all elements in a document) or very narrow (a single element). Rulesets define property groups. Depending on the property, only certain values and units are valid. Properties can do anything from adding space around an element to complex animations.

Learning Outcomes

When you've finished this lesson and its exercises, you should be able to:

  • Identify the parts of a CSS ruleset.
  • Apply styles to an HTML element using an element selector, a class selector, and an id selector.
  • Apply styles to an HTML element using child or descendent selectors.
  • Sketch the CSS box model.
  • Use CSS properties, values, and units.

CSS

CSS is not a programming language like Java, Python, or JavaScript, and it's not a markup language like HTML. CSS is a stylesheet language.

A stylesheet language defines rulesets that include a selector and a property set surrounded by curly braces. A ruleset may contain one to many properties.

CSS comments use the /* comment */ syntax.

/* ruleset */
html {
  font-family: Arial, sans-serif;
}

/* ruleset */
h2 {
  letter-spacing: 0.1rem;
  color: #0000aa;
  border-bottom: 1px solid #0000aa;
}

/* ruleset */
ul > li {
  list-style-type: "\1F47D";
}

Set Up

There are four ways to connect our CSS to HTML.

1. Embed directly in an element's style attribute.

<h1 style="border-bottom: 1px solid red;">Chapter 1</h1>
<p style="font-family: Arial, Helvetica, sans-serif; font-size: 1.2em;">
  It was a dark and stormy night.
</p>

In the strictest sense, this really isn't CSS. There are no selectors. It's one or more style properties connected directly to an element.

Never use this approach. It's an absolute nightmare to maintain. It's not DRY within the document and certainly not DRY in multiple documents.

2. Embed directly in a document's <style> tag.

CSS can be embedded directly in an HTML document using the <style> tag.

<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Inline CSS</title>
  <style>
    html {
      font-family: Arial, sans-serif;
    }

    h2 {
      letter-spacing: 0.1rem;
      color: #0000aa;
      border-bottom: 1px solid #0000aa;
    }

    ul>li {
      list-style-type: "\1F47D";
    }
  </style>
</head>

<body>
  <h1>Creamy Scrambled Eggs</h1>
    <h2>Ingredients</h2>
    <ul>
      <li>2 eggs</li>
      <li>1/4 cup heavy cream</li>
      <li>1 Tbsp butter</li>
      <li>Salt and pepper</li>
    </ul>
    <h2>Directions</h2>
    <ol>
      <li>Add eggs and cream to a small mixing bowl. Season to taste. Whisk briskly.</li>
      <li>Heat a small sauté pan over medium heat.</li>
      <li>Add butter to the pan. Swirl until it's melted, but don't let it brown.</li>
      <li>Add egg mixture to the pan. Turn gently. Cook to desired consistency. Don't over-mix. 
        This will cause the eggs to stick to the pan.</li>
      <li>Scoop cooked eggs on a plate alongside warm buttered toast.</li>
    </ol>
</body>

</html>

This approach isn't recommended. It violates the DRY principle. If we have many HTML documents, we would have to repeat our CSS in every document.

3. Link CSS externally via a relative URL.

A better approach is to link our CSS externally with a relative URL.

Box Model

Before we dive into CSS rulesets, we must understand the CSS Box Model. All HTML elements are bounded by a box. This means that a small image inside a link inside a navigation dropdown inside a navigation bar inside a body is a box in a box in a box in a box in a box. We need to understand these nested boxes to create sophisticated layouts.

Elements don't just have one box. They have four.

  1. Content box: HTML and text content.
  2. Padding box: a buffer between content and the element's border.
  3. Border box: the element's border. The edge of the border is the element's official boundary. We measure the width and height from the border.
  4. Margin box: a margin provides space outside of the element. The margin doesn't count against the element's width and height, but it does affect how elements are layed out.

Each HTML element has its own default padding, border, and margin (and font family, font size, etc). Those defaults usually aren't what we want. We can change the defaults with the padding property, the border property, and the margin property.

Both block-level and inline elements have boxes. Inline elements cannot be given a fixed width and height. They expand to fit their content. Inline elements also ignore vertical padding, borders, and margins. The padding, borders, and margin are there, but they don't change the vertical flow of an inline element.

Not to worry. We can always change a block-level element to inline and an inline element to block with the display property.

Selectors

A selector is a bit of syntax that starts a ruleset before its curly braces. A selector targets one or more HTML elements. It "selects" the elements from a crowd of many elements. Selectors can range from very simple to very complex.

/* generically:
some-selector {
  property: value;
}
*/

Element Selector

An element selector names the element's tag. Once named, the ruleset applies to all elements of that name. This isn't great for customizing a single element, but it's very helpful for setting a default style across an application.

/* generically:
element-name {
  property: value;
}
*/

p {
  padding: 5% 10%;
  margin: 12px;
}

Class Selector

If we want more control over our elements, say we don't want to select absolutely every element of a given type, we can use the class selector. A class selector begins with a dot (.) and ends with a class name.

/* generically:
.class {
  property: value;
}
*/

.medium-serif {
  font-family: Georgia, 'Times New Roman', Times, serif;
  font-weight: 700;
}

Each style-able HTML element may include an optional class attribute. A class is a name that's associated to a style. The class attribute may contain multiple names. The .medium-serif selector targets any element whose class contains the name "medium-serif".

<h2 class="medium-serif">Quote</h2>
<p class="medium-serif emphasis">
  "It's better to ask for forgiveness than for permission."
</p>

Id Selector

If we want even more control, we can use an id selector. An id selector targets a single element by its id value. It starts with a hash (#) and ends with the id of an element.

/* generically:
#some-id {
  property: value;
}
*/

#call-to-action {
  padding: 1.25rem;
  margin-top: 1.25rem;
  margin-bottom: 1.25rem;
  color: #087990;
  background-color: #cff4fc;
  border-left: 0.25rem solid #9eeaf9;
}

The selector #call-to-action targets the element with id="call-to-action". If there is no matching id, there's nothing to target.

Example

See the Pen CSS - id selector by Corbin March (@corbos) on CodePen.

Be careful with id selectors. When we target single elements, especially too many of them, it feels a bit like inline styles. Another selector may be more appropriate. For example, if there's only one h1 per page, use the h1 element selector. If there's any possibility of a second element on the page, use the class selector.

A Note On Naming

Use a functional and specific name for classes and ids.

Names cannot contains spaces. Use camel case (myClassName), dashes (my-class-name) or underscores (my_class_name) to separate words. Pick one and be consistent. (Though your client will probably already have a naming convention.)

Classes and ids are case sensitive. If we use a capital in a name, we must match the name in CSS.

Combinators (Combination Selectors)

Sometimes we don't want to give a class or id to every element to be styled. That's fussy and error-prone. Consider a set of links inside a navigation element. They should be styled the same, but different from other links on the page. It would be annoying (and, again, error-prone) to add a class to each navigation link.

Luckily, CSS provides combinators that define relationships between elements. Relationships include parent/child, grandparent/child, great-grandparent/child, (etc), and siblings.

Descendent Combinator

A descendent combinator creates a relationship between nested elements at any depth, not just parent/child. It starts with a selector for an ancestor element, followed by a space, and then a selector for descendent elements. The ruleset targets descendants.

/* generically:
ancestor descendent {
  property: value;
}
*/

nav a {
  color: green;
  text-decoration: underline;
}

.callout em {
  font-family: Verdana, Geneva, Tahoma, sans-serif;
  color: #cccccc;
}

Child Combinator

The child combinator selects first-level children of the parent element. It starts with a selector for a parent element, followed by a greater than symbol, >, and then a child element selector. The ruleset targets children.

/* generically:
parent > child {
  property: value;
}
*/

nav > a {
  color: green;
  text-decoration: underline;
}

.callout > em {
  font-family: Verdana, Geneva, Tahoma, sans-serif;
  color: #cccccc;
}

Sibling Combinators

The adjacent sibling combinator selects a sibling that immediately follows an initial sibling. It starts with a selector for the initial sibling, followed by the plus symbol, +, and then a selector for the trailing sibling. The ruleset targets the trailing sibling.

/* generically:
initial-sibling + trailing-sibling {
  property: value;
}
*/

.aside + p {
  background-color: khaki;
}

The general sibling combinator selects all siblings that follow an initial sibling. It starts with a selector for the initial sibling, followed by the tilde symbol, ~, and then a selector for all trailing siblings. The ruleset targets the trailing siblings.

/* generically:
initial-sibling ~ trailing-sibling {
  property: value;
}
*/

.aside ~ p {
  border: 1px solid #444444;
}

Selector List

If more than one selector has the same property set, we can group them in a single ruleset by separating selectors with commas.

Below, the first two rulesets are equivalent to the third ruleset.

/* These rulesets are equivalent...*/
label {
  font-weight: bold;
}

.disclaimer {
  font-weight: bold;
}

/* ...to this ruleset.*/
label, 
.disclaimer {
  font-weight: bold;

Values and Units

Each CSS property has a finite range of values and units it can accept. Intuitively, this makes sense. The font-size property shouldn't accept a color and the background-color property shouldn't accept a number.

CSS values have a value type (analogous to data type). They can be colors (red, #c55da1, rgb(197, 93, 161)), numbers (0.9, 12, -1.2), numbers with a unit (2.2rem, 10%, 70vh), URLs, an enumeration (solid, dashed, dotted), or more complicated value types.

Not all values have units. A unit is an absolute or relative measure of size.

Colors

Color values can be defined in six formats. We'll focus on three.

  • Name: a CSS named color -- "red", "aliceblue", "rebeccapurple".

  • Hex: a three-part hexadecimal value for the red/green/blue color components-- "#ff0000" red, "#f0f8ff" aliceblue, "#663399" rebeccapurple.

    This value: "#f0f8ff", breaks down to # (signifies a hex code) f0 (red saturation) f8 (green saturation) ff (blue saturation).

  • RGB: an explicit red/green/blue value ranging from 0-255 or a percent saturation -- "rgb(255,0,0)" red, "rgb(240, 248, 255)" aliceblue, "rgb(102, 51, 153)" rebeccapurple.

Here are common CSS properties that use color values.

.selector {
  color: thistle;
  background-color: #d8bfd8;
  border-color: rgb(216, 191, 216);
  box-shadow: 10px 5px 5px tomato;
  outline-color: #ff6347;
  text-shadow: rgb(255, 99, 71) 2px 5px;
  /*
  To make borders and outline visible, 
  we also need to add:
  border: 1px solid;
  outline: 1px solid;
  */
}

Color Accessibility

It's been a while since we mentioned accessibility. Web accessibility is just as important in CSS as it is in HTML.

Here are a few tips for strengthening accessibility with colors.

  • Black text on a white background is tiring to read. Soften the contrast.
  • Use the WebAIM contrast checker to confirm colors have enough contrast to be legible.
  • A significant number of people have trouble distinguishing color. Don't use color alone to indicate an error, warning, or notice.

Lengths

Many CSS properties accept lengths as value types. Examples include font-size, width, height, margin, and padding.

There are two types of lengths: absolute and relative. An absolute unit has a fixed size on the screen and when printed. A relative unit uses a font size, the viewport, or a container element to scale lengths up or down.

Absolute Units
Unit Example Description
px 24px A pixel. Depending on the screen, a pixel may or may not be an actual pixel. On large screens, a pixel is 1/96 of an inch.
in 2.5in An inch.
cm 2.5in A centimeter. 1cm == 0.394in
pt 12pt 1/72 of an inch.

The W3C discourages using in, cm, and pt for lengths. If we need an absolute unit, the pixel is preferred.

If at all possible, do not use pixel units for font-size. This is an accessibility problem. A relative unit allows users to adjust their font size.

Relative Units
Unit Example Description
em 1.2em

Ems are based on the font size of a parent element (historically, the width of capital "M"). They can be used for any length, not just font size. The numeric value is a multiplier. 0.5 is half of the parent's font width. 2.2 is two point two times the parent's font width.

When we have many nested parents, em can cause trouble. If we add a multiplier to each parent, the values compound. It's hard to keep track of reducing and widening multipliers.

rem 2.3rem Root Ems is the solution to compounding ems. It is based on the root element's font size. Again, the numeric value is a multiplier, but it's based on the root element, not all nested elements.
vh 25vh View height. The numeric value is a percent multiplier. 25 is a quarter of the view height. 10 is a tenth.
vw 30vw View width. The numeric value is a multiplier. 25 is a quarter of the view width. 10 is a tenth.

There are many other relative units. We can review them in the MDN length documentation. For now, we'll focus on pixels and relative ems.

Background Image Properties

We can set a background image in almost any HTML element.

CSS Properties that embed and manipulate background images include:

  • background-image, accepts an image URL. May also include linear gradient definitions.

  • background-repeat, valid values include:

    • repeat (tiles in all directions without gaps, may clip the image)
    • repeat-x (tiles horizontally without gaps, may clip the image)
    • repeat-y (tiles vertically without gaps, may clip the image)
    • no-repeat (doesn't tile, if the image is small, leaves gaps)
    • space (tiles in all directions, but if part of the image would be clipped, chooses not to clip and adds space around the tiles)
    • round(tiles in all directions, but if part of the image would be clipped, stretches to fill in the gaps)

  • background-position, valid values include:

    • top (aligns to the center top of the element unless modified)
    • left (aligns to the center left of the element unless modified)
    • right (aligns to the center right of the element unless modified)
    • bottom (aligns to the center bottom of the element unless modified)
    • center (aligns to the left/right, top/bottom center of the element unless modified)
    • The background-position property can also accept lengths and percentages.

Font Properties

Text content is arguably the most important content. Even with rich graphic design, text points us in the right direction and leaves breadcrumbs. Text is especially important for inclusive design. Screen readers rely on text to explore and navigate.

We can make our text beautiful and readable with CSS font properties.

  • font-family, accepts a comma-delimited list of font names. Multi-word font names, names with numbers, or names with punctuation must be surrounded with quotes. If a font isn't found, the next font in the list is used. Not all machines have all fonts so it's best to plan for a fallback. We can also explicitly include open web fonts with providers like Google Fonts.

    Always include a generic font family at the end of the list in case no matching font can be found. Generic families include:

    • serif: has characters (glyphs) that finish with a flare or taper.
    • sans-serif: has characters that finish with no flare or taper.
    • monospace: has characters with the same width.
    • cursive: has characters that look like cursive handwriting.

    There are other generic families, but the most common are serif, sans-serif, and maybe monospace.

  • font-size, accepts explicit sizes from xx-small to xxx-large, lengths (px, rem, etc), percentages (of the parent font size), and relative sizes (smaller, larger).

  • font-weight adds thickness to characters. Accepts normal and bold, lighter and bolder (relative to the parent), and numeric values 100, 200,... 900. Not all fonts support the shaded nuances of 100-900.

  • letter-spacing changes the whitespace between characters. Accepts normal and any length (px, rem, etc).

  • line-height adjusts the space between lines. Accepts normal, any length, percentages, and unit-less numbers that act as a multiplier for the element's font size.

Font Accessibility

Never embed text in an image. Screen readers can't read an image. Use text instead.

Avoid small font sizes.

Use a serif font when writing large bodies of text. The little feet guide the user's eye and make the site easier to read.

Box Model Properties

We mentioned the box model earlier. Let's dive into the details of CSS properties that support the box model.

  • padding creates space between an element's content box and border box. Padding makes space on the inside of the element. It accepts lengths and percentages.

    There are four forms:

    1. padding: value; -- applies to all four sides.
    2. padding: vertical horizontal; -- vertical adds space to the top and bottom, horizontal adds space to the left and right.
    3. padding: top horizontal bottom;
    4. padding: top right bottom left;

  • margin creates space outside the element's border box. It accepts lengths and percentages.

    There are four forms:

    1. margin: value; -- applies to all four sides.
    2. margin: vertical horizontal; -- vertical adds space to the top and bottom, horizontal adds space to the left and right.
    3. margin: top horizontal bottom;
    4. margin: top right bottom left;

  • border creates a border around an element. border is a shorthand property for three different properties: border-width, border-style, and border-color. Borders can be further broken down into specific sides with border-left, border-left-style, border-left-width, etc.

    • border-width (part of border) accepts either thin, medium, thick or a length.
    • border-style accepts none, hidden, dotted, dashed, solid, double, groove, ridge, inset, and outset.
    • border-color accepts a color.

    border has four forms:

    1. border: style;
    2. border: width style;
    3. border: style color;
    4. border: width style color;

    To gain more control of borders, be sure to review all border properties.

  • display changes the flow of an element in its layout and the flow of its child elements. If by default an element is block or inline, we can change it to inline or block.

    Common display values:

    • block: Change an inline element to a block element.
    • inline: Change a block element to an inline element.
    • inline-block: Change to an inline element but allow a bit of "block" behavior. We can apply an explicit width and height to inline block elements.
    • flex: Activates a flexbox layout. This is an advanced use case. We won't cover it in this lesson.
    • grid: Activates a grid layout. This is an advanced use case. We won't cover it in this lesson.
    • table: Mimics an HTML table. This value isn't used often because we can use a grid layout.
    • none: Don't display the element. Removes it from the flow.

    There are many other display values (inline-flex, inline-grid, table-cell, etc). We won't cover them in this lesson.
