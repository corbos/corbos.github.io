Gomoku

Gomoku, or Five in a Row, is a game traditionally played on a Go board. Players take turns placing stones on the board to form a connected sequence of exactly 5 stones horizontally, vertically, or diagonally. A standard board is 15 X 15. Players must balance their offensive strategy (creating a 5 stone sequence) with their defensive strategy (preventing their opponent from creating a 5 stone sequence). The first player to place five stones in a row wins.

Rules

  • Stones are black and white.
  • Black stone plays first.
  • Players alternate turns.
  • Players can play anywhere on the board.
  • The first player to get exactly 5 stones in a row wins. 6 or more stones in a row, called an overline, is not a win.
  • Draws are possible.

Your Code

Your code should control game-play, display the state of the game, and accept user input for human players.

Add as many modules, classes, functions, and methods as you see fit to build a fun and easy-to-use console interface.

Requirements

  • Set up two players.
  • For a human player, collect their name. (A random player's name is randomly generated.)
  • For each stone placement, use the player's name to prompt for row and column.
  • Since the random player doesn't require input, the UI should display stone placement and the results of placement. (Random player placement may fail since they don't know what they're doing.)
  • Re-prompt on failed placement. The game must not proceed until a player has made a valid move.
  • Display the final result of the game.
  • Give the option to play again.

Technical Requirements

  • All rules are modeled inside the gomoku.py module.
  • Be sure to use sensible module, class, function, and method names.
  • Stones use 0-based rows and columns.

Approach

Plan before you write code.

It may be helpful to break the game into two phases: set up and game-play.

Write one function/method that accepts the game state and draws the board. Never draw the board in more than one place.

Stretch Goals

  • Create a third implementation and add it to your list of player options. Can you build a player that makes good choices?
  • Black has an advantage as the starting player. Competitive Gomoku uses various opening rules to improve fairness. Implement one of those rules. In the starting screen, give the option to choose an opening rule.
